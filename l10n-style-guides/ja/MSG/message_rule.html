メッセージファイルのアップデート・編集 (抜粋)

エンジニアからの指示にしたがって、該当するファイルを編集します。

翻訳対象のメッセージファイルの種類

  • .po
  • .msg
  • .tmsg
  • .java
  • .properties
  • .info
  • オンラインヘルプ
    • .html
    • *.hs
    • *MAP.jhm
    • *TOC.xml
    • .help、.hlp
    • 拡張子なし

翻訳・アップデートを行う際の注意事項

各メッセージファイル形式に共通

  • SCCS 管理のファイルをアップデートする場合は、どの build の変更を反映したのかがわかるように、SCCS delget 時のコメントに記入すること。

    • 例:
    % sccs delget <filename>
    comments?  [s8_23] incorporated base changes.
    bringover, putback, SCCS ファイル編集の方法
  • 翻訳対象外の文字列

    • 変数:  %s (文字列)、%c (文字)、%d (10 進数) など。
  • 変数の扱い

    • 日本語メッセージでも、変数の順番が英語と同じになるようにすること。
    変数の順番を変えなければ日本語にできない場合は、次のようにして変数の順番を示す数値を指定すること。
    例:  msgid   "Checking uninstalled directory format package"
          msgstr  "<%2$s> からアンインストールされたディレクトリ形式"
          msgid   "<%s> from <%s>＼n"
          msgstr  "パッケージ <%1$s> を検査中です。＼n"

.po ファイル

必要な作業

変更があった msgid の行を最新の英語版と同じにする (マージ)
対応する msgstr の行を日本語に翻訳する。

コメントアウト行

行頭に # がある行は、メッセージとして認識されないコメント行です。

エスケープシーケンス

バックスラッシュ ＼ でエスケープする。
例: 二重引用符 " をエスケープする場合
use ＼"ab2admin -o start＼" to....    のようにする。

.msg ファイル

必要な作業

英語メッセージを翻訳した日本語メッセージに置き換える。
メッセージ番号とメッセージ文字列との間に必ず半角スペースを 1 つ入力すること．
　例:　3  スーパーユーザーのパスワードを入力してください。＼n

改行

メッセージを複数行に改行するには、＼n を入力します。

コメントアウト行

行頭に $ がある行は、メッセージとして認識されないコメント行です。

.tmsg ファイル

必要な作業

英語メッセージを翻訳した日本語メッセージに置き換える。
メッセージ ID 番号とメッセージ文字列との間に必ず半角スペースを 1 つ入力すること。
　例:　32134  root で admintool を起動

改行

メッセージを複数行に改行するには、＼＼＼n＼ を入力します。
　例:   32400  このアイコンは、ブロック型特殊ファイルを表しています。＼＼＼n＼
                                データ型は BLOCK_DEVICE です。

コメントアウト行

行頭に $ がある行は、メッセージとして認識されないコメント行です。

.java ファイル

必要な作業

・ { "MsgID" ,  "Message xxxxxxxxx" },  の Message xxxxxxxxxx の部分を日本語メッセージに翻訳する。


・ ファイル名と、各クラス名に _ja を付ける。
        たとえば "AddressSyncConduitMessages.java" の場合は、
        ファイル名を "AddressSyncConduitMessages_ja.java" にして、

        public class AddressSyncConduitMessages extends ListResourceBundle という行を
        public class AddressSyncConduitMessages_ja extends ListResourceBundle にする。

改行

メッセージを複数行に改行するには、+ を入力します。
例 :    { "MSG8", "製品が正しく動作するためには"  +
           " システムをリブートする必要があります。 " +
           " ここでリブートしますか ?"},

コメントアウト行

行頭に // がある行、または先頭 /** と末尾 */ で囲まれている行は、メッセージとして認識されないコメント行です。

.properties ファイル

必要な作業

tag=messages の "messages" 部分を日本語メッセージに翻訳する。
   例: init_view_explain=Choose what to display for this first time request.
                         ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
                              ここを翻訳↑

改行

メッセージを複数行に改行するには、＼n を入力します。

コメントアウト行

行頭に # がある行は、メッセージとして認識されないコメント行です。

ヘルプファイル

必要な作業

英語の文章を翻訳した日本語文章に置き換える。

.html ファイル

<meta http-equiv="Content-Type" content="text/html; charset=iso-8859-1">
                                                            ~~~~~~~~~~~
                                                            euc-jp にする。

*.hs、*MAP.jhm、*TOC.xml ファイル

.html ファイルと共に、*.hs、*MAP.jhm、*TOC.xml ファイルが含まれていることがある。
  • *.hs
    先頭行 <?xml version='1.0' encoding='ISO-8859-1' ?> を <?xml version='1.0' encoding='EUC_JP' ?> にする。
    <title> および <label> タグで囲まれた部分を日本語に訳す。
  • *MAP.jhm
    先頭行 <?xml version='1.0' encoding='ISO-8859-1' ?> を <?xml version='1.0' encoding='EUC_JP' ?> にする。
    <tocitem text= " xxxxxx " target=" "/> の xxxxx 部分を日本語に訳す。
  • *TOC.xml
    先頭行 <?xml version='1.0' encoding='ISO-8859-1' ?> を <?xml version='1.0' encoding='EUC_JP' ?> にする。

.help、.hlp ファイル

行末をそろえる (1 行の文字数を半角 50 字くらいにする)

topics/TOPICS ファイル

以下のように、ファイル名の行の先頭に、スペースではなくタブで空白を入力すること。
(そうしないと、実際にヘルプを表示する時にトピックスとして認識されない)
例:
DNS 検索リスト
(TAB) dns_searchdomains.help
DNS サーバー
(TAB) dns_server.help

拡張子なしファイル

行末をそろえる (1 行の文字数を半角 28 文字くらいにする)
